# Engineering PR / DoD Templates v0

## 1) PR 템플릿

### 배경
- 어떤 문제를 해결하는지
- 왜 지금 필요한지

### 변경점
- 핵심 변경 3줄 요약
- 영향 컴포넌트(backend/frontend/db)

### 검증
- 로컬 검증 명령
- 테스트 결과 요약
- 수동 검증 시나리오

### 마이그레이션/롤백
- 스키마 변경 여부 (있음/없음)
- 적용 명령 (예: `npm run migration:run`)
- 롤백 절차 (예: `npm run migration:revert` 또는 배포 롤백)

### 리스크
- 기능/성능/보안 리스크
- 모니터링 포인트

## 2) 작업 티켓 DoD 템플릿
- 요구사항이 코드/문서에 반영되었다.
- 빌드가 성공한다.
- 테스트 또는 수동 검증 결과가 코멘트에 남아 있다.
- 롤백/운영 반영 절차가 명시되어 있다.
- 관련 이슈 링크가 코멘트에 포함되어 있다.

## 3) 리뷰 체크리스트 (최소 기준)
- 요구사항 대비 누락 없음
- 오류 처리/예외 경로 확인
- DB 변경 시 마이그레이션 안전성 확인
- 로그/모니터링 포인트 확인
- 보안 민감정보 노출 여부 확인

## 4) 샘플 적용
샘플 이슈: [EZC-31](/EZC/issues/EZC-31)

- 배경: PR/DoD 기준 부재로 리뷰 품질 편차 발생
- 변경점: 템플릿 초안 문서화 및 체크리스트 정의
- 검증: 문서 리뷰(CTO) 완료
- 롤백: 문서 변경이므로 코드 롤백 불필요
- 리스크: 템플릿 과도화 시 초기 속도 저하 가능
