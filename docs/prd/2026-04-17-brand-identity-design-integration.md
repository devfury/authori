# PRD: Authori 브랜드 아이덴티티 및 디자인 통합 가이드

## 1. 개요 (Overview)
Authori 플랫폼의 시각적 정체성을 확립하기 위해 '오소리(Badger)' 캐릭터와 '보안(방패, 열쇠, 포털)' 테마를 서비스 전반에 통합합니다. 이를 통해 사용자에게 신뢰감 있고 친근한 보안 경험을 제공하는 것을 목표로 합니다.

## 2. 디자인 컨셉 분석
*   **핵심 캐릭터:** 오소리 (끈기 있고 강인한 보호자의 이미지)
*   **주요 오브젝트:**
    *   **방패 (Shield):** 데이터 보호 및 보안의 안정성
    *   **열쇠 (OA Key):** 인증 및 접근 권한의 핵심 (OAuth의 'OA' 상징)
    *   **포털 (Portal):** 안전한 서비스 진입로
*   **메인 컬러 팔레트:**
    *   Primary: Deep Blue (신뢰, 전문성)
    *   Secondary: Gold Yellow (핵심 자산, 강조)
    *   Background: Light Blue/White (명료함, 청결함)

## 3. 주요 반영 영역 (Core Integration Areas)

### 3.1. 브랜드 및 공통 요소
*   **Favicon & Logo:** `frontend/public/` 내 로고를 오소리 캐릭터와 방패 결합형 아이콘으로 교체.
*   **Global Style:** `frontend/src/style.css`에 브랜드 컬러 변수 정의 및 적용.

### 3.2. 사용자 경험 (UX) 포인트
*   **로그인/로그아웃 페이지:** 
    *   포털 앞에 서 있는 오소리 일러스트 배치.
    *   로그인 성공 시 포털이 활성화되는 애니메이션 효과.
*   **상태 페이지 (Error/Loading):**
    *   404 페이지: 길을 찾는 오소리 캐릭터 활용.
    *   로딩 스피너: 열쇠를 돌리거나 포털 주변을 뛰는 오소리 애니메이션.
*   **온보딩 가이드:** 오소리 캐릭터를 가이드 봇으로 활용하여 초기 설정 안내.

### 3.3. 기능별 디자인 요소
*   **보안 설정 및 키 관리:** API 키 생성 시 'OA 열쇠' 아이콘 활용.
*   **대시보드:** 보안 점검 결과에 따라 오소리의 상태(방패를 든 모습 등)를 시각적으로 노출.

## 4. 기술적 구현 계획 (Technical Plan)
1.  **에셋 최적화:** 제공된 이미지를 SVG 및 투명 PNG로 분리하여 `frontend/src/assets/brand/`에 관리.
2.  **테마 시스템:** CSS Variables를 사용하여 다크/라이트 모드 대응 시 브랜드 컬러 유지.
3.  **컴포넌트화:** 캐릭터 일러스트를 재사용 가능한 Vue 컴포넌트(예: `AuthoriBadger.vue`)로 제작.

## 5. 기대 효과
*   브랜드 인지도 향상 및 경쟁 서비스와의 차별화.
*   딱딱한 보안 서비스의 이미지를 탈피하여 사용자 친화적인 접근성 확보.
*   일관된 시각 언어를 통한 플랫폼 신뢰도 강화.
